# 仕事に役立たないHTMLの雑学 アウトライン

- 仮タイトル: 仕事に役立たないHTMLの雑学
- サブタイトル: なぜHTMLはこうなったのか

## この本の目的

- 読者に理解してほしいこと:
  - HTML は「変な技術の寄せ集め」ではなく、文書共有、互換性、実装現実の折衷として育った技術であること。
  - ブラウザが HTML を補完し回復する理由は、甘さではなく Web の責務に根ざしていること。
  - 仕様書の長さや古い要素の残存には、歴史的・制度的な理由があること。
- 読者に判断できるようになってほしいこと:
  - 実務でよく聞く HTML の通説が、どの条件では正しく、どこから雑になるかを判断できること。
  - View Source と DevTools、仕様と実装、歴史的事情と現代の推奨を切り分けて説明できること。
- この本が目的にしないこと:
  - HTML タグの網羅的な入門
  - フォームやレイアウトの手取り足取り解説
  - アクセシビリティや SEO の包括的な教科書

## 想定読者

- 前提知識:
  - HTML を書いたことがある
  - CSS と JavaScript が HTML と分業していることは知っている
- 想定経験:
  - Rails アプリやフロントエンド実装で HTML を触ったことがある
  - `tbody` や `h1`、`div`、`table` について断片的な通説を聞いたことがある
- つまずきやすい点:
  - 仕様上の規則と実務の助言が混ざる
  - View Source と DOM の違いが曖昧
  - `HTML5` と現行仕様の関係が分からない
