# 付録A 明日話したくなるHTML雑学50

1. `tbody` は書かなくても DOM に現れることがあります。
2. `html` `head` `body` は省略しても補完されます。
3. `p` 要素は文脈によって暗黙に閉じられます。
4. View Source と DevTools は同じものを見ていません。
5. HTML 仕様書のかなりの部分はパーサーの規則です。
6. `img` は最初から子を持たない要素です。
7. `br` も `hr` も空要素です。
8. コメントにも書いてはいけない並びがあります。
9. ブール属性は値を書かなくても成立します。
10. 引用符は省略できる場面がありますが、いつでもではありません。
11. HTML はプログラミング言語ではありません。
12. HTML は URL や HTTP と分業して Web を作ります。
13. 世界初の Web サイト `info.cern.ch` は今も公開されていて、当時のページを見られます。
14. ブラウザは壊れた入力を止めずに回復します。
15. 1989 年の Web 提案書には、上司が「Vague but exciting（曖昧だが、わくわくする）」と書き込みました。
16. `table` は悪ではなく、用途の誤用が問題でした。
17. `div` は意味がないのではなく、汎用要素です。
18. `b` は完全に消えた要素ではありません。
19. `strong` と `b` は同じではありません。
20. `h1` は仕様上 1 個だけとは限りません。
21. ただし見出し階層は論理的である必要があります。
22. XHTML は厳格さを目指しましたが、Web の現実と衝突しました。
23. XML の「壊れたら止める」は Web では重すぎました。
24. Quirks Mode は古いページを守るために残っています。
25. 古いページが今も開けるのは偶然ではありません。
26. HTML の歴史には presentational な要素の時代がありました。
27. `font` や `center` は obsolete でも、当時の需要は本物でした。
28. obsolete は「なかったこと」ではなく、歴史の層として残ります。
29. 仕様は理想論だけでなく実装の一致も目的にしています。
30. WHATWG の HTML は Living Standard です。
31. 仕様は版で止まるより更新され続けます。
32. ブラウザ戦争は標準化の必要性を強くしました。
33. HTML の読者はブラウザだけではありません。
34. 検索エンジンも HTML の構造を読みます。
35. スクリーンリーダーも HTML の意味づけに依存します。
36. 最初の Web サーバーになった NeXT には「電源を切るな」という貼り紙がありました。
37. 空要素は「閉じ忘れ」ではありません。
38. 属性の省略可能性には SGML の名残があります。
39. HTML は簡単に見えて、仕様はかなり大きいです。
40. それはブラウザ実装の現実を含むからです。
41. バグが互換性の一部になることがあります。
42. `img` タグは 1993 年にマーク・アンドリーセンが提案したものです。
43. HTML の長寿は後方互換性に支えられています。
44. Netscape の `<blink>` は一晩で実装されたと言われ、のちに各ブラウザから削除されました。
45. 文字を流す `<marquee>` は Internet Explorer の独自タグでした。
46. フレームワークが変わっても最終的に HTML へ落ちることは多いです。
47. W3C と WHATWG は 2019 年に HTML 標準を WHATWG の Living Standard へ一本化しました。
48. HTML は完成しない仕様です。
49. `<details>` と `<summary>` だけで、JavaScript なしに開閉する UI が作れます。
50. HTML の奇妙さの多くには歴史的な理由があります。
