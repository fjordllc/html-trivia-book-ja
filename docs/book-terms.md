# HTML雑学本 用語メモ

このファイルは、『仕事に役立たないHTMLの雑学』で表記を揺らさないための補助メモです。FBC Press 全体ルールに追加して、この本で頻出する語の扱いを固定します。

| 採用表記 | 使わない表記 | 補足 |
| --- | --- | --- |
| HTML仕様書 | HTML spec、仕様 | 本文では `HTML仕様書` を優先 |
| ブラウザ実装 | 実装、ブラウザ側実装 | 必要なら `ブラウザ実装` と明示 |
| パーサー | parser | 初出では `パーサー（構文解析器）` 可 |
| エラー回復 | エラー耐性、回復処理 | HTML 文脈では `エラー回復` |
| 後方互換性 | 互換性 | 一般互換性と区別したいときは `後方互換性` |
| 支援技術 | アシスティブ技術 | 本文では `支援技術` を優先 |
| 空要素 | 自己終了タグ | HTML 文脈では `空要素` が正確 |
| 都市伝説 | 誤解、俗説 | 付録C と整合させて `都市伝説` を採用 |
| Living Standard | 生きた標準 | 固有名として英語表記を残す |

## 運用メモ

- `HTML5` は歴史的文脈に限って使い、現行仕様全体を指すときは `HTML Living Standard` または `現行の HTML 仕様` と書き分ける。
- `DOM` は初出で `DOM（文書オブジェクトモデル）` と補足してよい。
- `View Source` と `DevTools` は UI 上の表記としてこのまま使う。
