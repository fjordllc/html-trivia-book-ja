# 9ロールレビュー記録

対象: `manuscript/` 配下の全原稿  
基準: `../FBC_Press/STYLE_GUIDE.md`, `REVIEW_CHECKLIST.md`, `CHAPTER_TEMPLATE.md`

## 総評

企画の軸はぶれていません。HTML を笑いものにせず、「なぜそうなったのか」を歴史・仕様・実装の交点で語ろうとしている点は、FBC Press の思想と整合しています。

一方で、現段階の本文は「章の核となる説明を短く書いた下書き」に近く、FBC Press が要求する節粒度、判断基準、一次情報の明示、誤解の先回りまで届いていません。したがってレビュー結論は「企画は良いが、本文は改稿前提」です。

## 役割別レビュー

### 1. HTML博士

- 良い点: 題材選定が的確です。`tbody`、`p`、`img`、`b` など、雑学として知られやすいが本質的な論点を選べています。
- 懸念: 仕様用語の厳密さが足りません。`空要素`、`ブール属性`、`セクショニング`、`アウトラインアルゴリズム` のような語が、章によっては短く流れすぎています。
- 優先修正: 各章で 1 回は仕様用語をきちんと定義し、その用語がどの規則に紐づくかを示す。

### 2. ブラウザ開発者

- 良い点: ブラウザの補完やエラー回復を「甘さ」ではなく責務として扱っている。
- 懸念: パーサーや DOM 構築の因果がまだ薄い。`何が起きるか` は書いてあるが、`どの規則でそうなるか` が弱い。
- 優先修正: 第4〜8章で、入力文字列、パース、DOM の三層を毎章明示する。

### 3. HTML仕様策定者

- 良い点: 都市伝説を避ける姿勢は適切です。
- 懸念: `HTML5` の歴史的文脈と、現行の `HTML Living Standard` の文脈が混ざりやすい。仕様上の規則と実務上の助言も分離が弱い。
- 優先修正: 章末参考資料で仕様と解説を分ける。本文中でも「仕様上」「実務上」を書き分ける。

### 4. コンピュータ史研究者

- 良い点: Tim Berners-Lee、ブラウザ戦争、XHTML など、歴史の節目は外していない。
- 懸念: 年代感が薄く、歴史が「今の結論のための前置き」に見える章がある。
- 優先修正: 第2章、第13〜16章、第21〜23章では、少なくとも時代背景と転換点を一段深く書く。

### 5. 技術書編集者

- 良い点: 本全体の流れは分かりやすく、部構成も自然です。
- 懸念: 各章に節番号がなく、参照性が低い。章内の起伏も弱く、読者がどこを読み返せばよいか分かりづらい。
- 優先修正: 各章を最低 4〜6 節に分ける。`導入 → 具体例 → なぜ → 誤解 → まとめ` の型を機械的にでも入れる。

### 6. Railsエンジニア

- 良い点: 想定読者に合う話題が多い。View Source と DevTools の違い、h1 論争、table/div の話は実務に引き寄せやすい。
- 懸念: 「この知識が日々の実装でどう役立つか」が少し弱い。
- 優先修正: 各章末で、実務では何を気にすべきかを一段入れる。

### 7. 辛口Amazonレビュワー

- 良い点: テーマそのものは興味を引く。
- 懸念: 今のままだと「薄い」「ブログ連載をまとめたようだ」「参考資料が弱い」と批判されやすい。
- 優先修正: 章密度を上げる。一次情報リンクを具体化する。章ごとの差別化を強める。

### 8. 著者本人

- 良い点: HTML への敬意は出ている。
- 懸念: 章によってはまだ「理由の入口」で止まっている。もっと腑に落ちるところまで掘れる。
- 優先修正: 各章で「なぜそうなったのか」の中心説明を 1 段深くする。

### 9. FBC Press編集長

- 良い点: 企画意図、想定読者、本全体の軸は合格です。
- 懸念: 本文がまだ FBC Press の完成原稿密度ではない。構成テンプレ、判断基準、参考資料の精度が不足。
- 優先修正: 節番号、判断基準、参考資料の 3 点を最低ラインとしてそろえる。

## 部ごとの所見

### 第1部

- 強み: 本の入口として自然です。
- 弱み: 第1章〜第3章ともに、背景説明はあるが「従来は何だったか」「なぜ HTML が採られたか」の比較がもう一段欲しい。

### 第2部

- 強み: 本書の山場になりやすい題材がそろっています。
- 弱み: 実装と仕様の説明密度が足りず、雑学から一歩先の理解へ届ききっていません。

### 第3部

- 強み: 設計思想を扱う題材選びは良いです。
- 弱み: 第9〜12章は特に短く、章というより節の長さです。

### 第4部

- 強み: 理想と現実のあいだという部テーマは明快です。
- 弱み: 歴史叙述の厚みがまだ不足しています。

### 第5部

- 強み: 読者が引っかかりやすい論争点を的確に押さえています。
- 弱み: 判断基準は出ているが、条件分けがまだ粗いです。

### 第6部

- 強み: 人と制度の話を後半に置いた構成は効果的です。
- 弱み: 章同士の差がまだ弱く、歴史章としての具体が足りません。

### 第7部

- 強み: 締め方の方向は良いです。
- 弱み: 結論部としては、前章までを束ねる再統合が薄いです。
